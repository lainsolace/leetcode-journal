# LeetCode Journal

This repository serves as a comprehensive log of my LeetCode problem-solving journey. It's a place to document problem analysis, exploration of multiple solutions, and the thought process behind arriving at optimal approaches.

## 📂 Directory Structure

Each problem is organized into a dedicated directory following the naming convention: `problem_number_title`.

For example: `1_two_sum`

Within each problem directory, you will find:

- `README.md`: Detailed problem analysis, constraints, and examples.
- `solution_1.md` - `solution_n.md`: Multiple solutions with explanations, time/space complexity analysis, and implementation details.

## 📝 How to Use

- **Review**: Explore the `README.md` to understand the problem.
- **Study**: Dive into the `solution_*.md` files to learn different approaches.
- **Learn**: Focus on the explanations and complexity analysis to grasp the core concepts.

## 🎯 Goals

- Document solutions for all major LeetCode topics.
- Explore and compare multiple approaches for each problem.
- Maintain clear and concise explanations for each solution.
- Track progress and learning milestones.
